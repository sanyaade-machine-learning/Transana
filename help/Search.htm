Search

To search for a single Keyword, you can right-click a Keyword in the Data window's database tree and select Quick Search from the popup menu. A node named after the Keyword Group : Keyword pair will appear under the Search node containing a tree structure with all Quote, Clip, and Snapshot records that have the selected Keyword.

If you want to do a more complex search, right-click on the Search root node on the database tab of the Data window and select Search to search your Documents, Episodes, Quotes, Clips, and Snapshots for the presence of assigned Keywords. The Search dialog will appear.

The Keywords Tab of the Search dialog box has been carefully designed to only allow valid entries and to produce valid searches. If something you want to choose is not available, it is because you have missed something required to make your specification complete. For example, if the "Search" button is not enabled, it is most likely because you have not closed a set of parentheses or because you just selected a logical operator and have not yet specified the next Keyword Group : Keyword pair.

The Collections tab of the Search Dialog allows you to specify what Collections Transana should include or exclude during a Search. This allows you to limit your Search Results to only a subset of your analysis, should you wish. This helps Collections behave more like Keywords in Transana as far as Search is concerned.

When you have constructed a valid search query, you can save it for later reuse by pressing the  icon in the lower left corner of the dialog. You can then load the saved search later using the  icon or delete it using the  icon.

An Important Note about Searches

The Search dialog box allows you to specify complex searches, including using boolean logical operator (AND, OR, and NOT,) but you must do so carefully to be sure to get the results you intend.

For example, the searches

  • (A or B) and C
and
  • A or (B and C)
produce different results.

The first search will find all Quotes, Clips, and Snapshots that have either Keyword A or Keyword B which also have Keyword C, producing hits on combinations AC, BC, and ABC.

The second search will find all Quotes, Clips, and Snapshots that have Keyword A, as well as those that have both B and C, producing hits on A, AB, AC, BC, and ABC. These are obviously very different results, with the second search including more results.

You should be particularly careful with something like the more ambiguous search

  • A or B and C.
It's the same search terms in the same order as the two searches described above, which produced different results. So what results should you expect?

Many people initially expect this last search to be equivalent to the first query above. However, this search is actually equivalent to the second search described above, and will find all Quotes, Clips, and Snapshots that have Keyword A, as well as all those that have both B and C. This is because AND is higher than OR in the Boolean "order of operations" and is therefore evaluated first by the Search engine.

To avoid problems, liberal use of parentheses is recommended.
