Snapshot Properties

Please note that the Standard version of Transana does not support Snapshots. To use this feature, you must be using the Professionsal, Computer Lab, or Multi-User version of Transana.

A Snapshot in Transana is a basic analytic unit for still images. The Snapshot is not a smaller segment of the still image the way a Quote is a segment of a Document or a Clip is a segment of an Episode. A Snapshot is created directly in a Collection. The relationship between detail and the larger image is handled by the Snapshot Window rather than through the Data Window.

To add a Snapshot in Transana, you can right-click (Ctrl-click on the Mac) on the Collection where you want to place the Snapshot and select "Add Snapshot" from the popup menu. Doing this will add the Snapshot to the end of the list of data objects in the Collection.

If you want to place the new Snapshot in a particular position in the list of data items in a Collection, you can right-click a Quote, Clip or Snapshot and choose "Add Snapshot,". If you do this, the new Snapshot will be added above the selected item.

You will then see the Add Snapshot dialog box.

Press the Browse button to select an image file from the computer's disk. If you currently have an Episode or Clip displayed in Transana's main interface, you can also press the "Capture Snapshot for Coding" button right next to the Browse button to capture a still image from the currently-loaded video file. (If you don't have an episode or clip loaded in Transana's main interface, this option is not visible.)

Please note that Transana does not import the image file into the database, but rather stores a pointer to the file's location. If you rename, move, or delete the image file after creating a Snapshot, that Snapshot will not load properly. If you capture a Snapshot from your video, you should avoid using the default Snapshot name to avoid accidentally deleting the image file created in this process should you capture another Snapshot from the same video file at a later time.

The only required fields are the "Snapshot ID," which must be unique within the Collection, and the Image Filename. If you would like to link your Snapshot to a particular spot in a media file, you can use the "Library ID", "Episode ID", "Transcription ID", "Episode Position" and "Duration" fields. "Comment" can be used in any way you wish.

You can apply coding to an entire still image by selecting a Keyword Group and Keyword on this form, then clicking the  button. This allows you to find this Snapshot later when doing Keyword searches. The  button allows you to delete Keywords from the Snapshot Keyword list, and the  button allows you to access Keyword Management to add or edit keywords while in the middle of adding a Snapshot. The "Collection ID" field is maintained by Transana and cannot be edited. (You can also code parts of an image in the Snapshot Window.)

One final note: Unlike other branches of the database tree, items within a Collection have an "order" that can be altered by the user. (This is important for the Collection Report and "Play All Clips".) If you choose "Add Snapshot", your new Snapshot will be added to the end of the list of items within the Collection. However, if you choose "Add Snapshot" on an existing Quote, Clip, or Snapshot, the new Snapshot will be inserted in the list above the item you selected. The order of items within a Collection can be altered using cut-and-paste or drag-and-drop within the Collection.
